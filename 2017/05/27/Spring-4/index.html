Spring(四)基于 xml 配置 bean(二)

bean 的继承

使用 parent 属性，parent 的值为被继承的 bean 的 id 值
注意：

  • 子 bean 可以重写属性值；
  • 父 bean 既可以作为配置模板，也可以作为 bean 实例，若父 bean 只作为模板，可将 bean的 abstract 属性设置为 true，这样 Spring 将不会实例化这个 bean；
  • 并不是 bean 里的所有属性都会被继承，如 autowire、abstract等；
  • 当忽略父 bean 的 class 属性时，abstract 必须设为 true；

bean 的依赖

使用 depends-on 属性设置前置依赖的 bean
注意：

  • 前置依赖的 bean 会在本 bean 实例化之前创建好；
  • 若依赖于多个 bean，可用逗号或空格隔开；

bean 的作用域

使用 scope 属性设置
注意：

  • singleton：默认值，容器初始化时创建 bean 实例，在整个容器的生命周期内只创建一个 bean，单例的；
  • prototype：原型的，容器初始化时不创建 bean 实例，而在每次请求时都创建一个新的 bean 实例，并返回；

bean 使用外部属性文件

  • 外部属性文件适用于：系统部署的细节信息（如文件路径、数据源配置信息等）；
  • 需要导入 context 的命名空间；
  • 由 PropertyPlaceholderConfigurer 的 BeanFactory 后置处理器完成，使用形式为 ${var}；
